CT-Monitor for Solitary Survival Strategy 2030
1. 概要
本プロジェクトは、既存の見守りサービスで用いられるスマートメーターの「Bルート」等のデータが抱える「インバーターエアコン等の低消費電力ノイズによる安否誤判定」という欠陥を克服するため、分電盤や天井内での直接電流計測（CT方式）を採用した実務特化型監視装置である。

2. 開発の背景（既存技術の限界）
既存の電力系見守り特許（電力会社・NTT等）は、スマートメーターから得られるマクロなデータを「演算」で解析しようとするが、現場では以下の理由により機能しない可能性がある。

エアコンのインバーター制御: 設定温度付近での微細な電力変動は、生存確認のノイズに埋もれる。
「エアコンつけっぱなし」での死亡: 電力消費が継続するため、既存ロジックでは「生存」と判定され、死後放置（汚損損害）を招く。

本装置は、分電盤の特定回路（照明やコンセント）を直接計測することで、これらの「推測」を排除し、「物理的な活動事実」を確定させることを目的とする。

3. ハードウェア構成（標準的かつオープンな選定）
本構成は汎用部品の組み合わせであり、特定の企業による独占を拒絶する。

CTセンサー: SCT-013-000 等（クランプ式・非破壊設置）
ADC: ADS1115（16bit高精度アナログ-デジタルコンバータ、I2C接続）等
MCU: ESP32 / ESP32-C6 等（BLE / ESP-NOW / Wi-Fi搭載）
電源部: Deep Sleepロジックを用いた電池駆動、または絶縁型AC-DCモジュール。

上記のシステムを分電盤もしくは天井内に設置し、電線にクランプする。

4. 独自の運用アルゴリズム
Deep Sleep & Interval Measurement: 1分に1回（任意設定可能）のみ起動し、サンプリング後に即座にDeep Sleepへ移行。
これにより、１分以上連続動作が発生する電子レンジや照明回路など生活にて使用する電気器具の使用を監視する。

ESP-NOW / BLE パケット通信: ペアリングや複雑なハンドシェイクを排除した軽量プロトコルにより、消費電力を極限まで抑え、分電盤や天井内からの安定した送信を実現。

親機連動型死活監視: 60分以上パケットが到達しない場合、親機側で「通信障害」として即座にアラートを発火。
